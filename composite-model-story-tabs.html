Composite Model over a Published Model

How the situation was created — and what every Copilot question pays for it

DEVELOPER A builds and publishes the original model
☁️ Azure Synapse
Enterprise data source. Developer A connects in Import mode.
👤 Developer A — Power BI Desktop
Builds tables, relationships, DAX measures → publishes to the Premium workspace.
🧊 Semantic Model A IMPORT
Data compressed into VertiPaq memory on the capacity. Scheduled refresh from Synapse. RLS defined here flows downstream.
DEVELOPER B Contributor + Build permission on Model A
👤 Developer B — Power BI Desktop
Live-connects to Semantic Model A, then adds local tables. The moment tables are added, the live connection silently converts to DirectQuery over Analysis Services — it's now a composite model.
🔗 Semantic Model B NEW · COMPOSITE
Publishing creates a brand-new semantic model — Model A stays untouched. Model B stores Developer B's tables plus a metadata-only link to Model A. No data is copied.

⚠ The consequence

Two models now live on the capacity instead of one. Model B depends on Model A — structural changes or permission changes in A break B. And every report interaction or Copilot question on Model B queries Model A live, every time. See Tab 2 for what that journey looks like →

🙋 Copilot User
Viewing a report built on Model B. Types one natural-language question.
🤖 Copilot (Fabric)
One question becomes multiple DAX queries: ① inspect schema  ② sample column values  ③ run the answer query.
🔗 Semantic Model B COMPOSITE
Local tables: answered here instantly, in memory.
Model A's tables: only metadata lives here — no data. Those queries must travel onward.
🧊 Semantic Model A IMPORT
Executes each incoming query in its own VertiPaq memory — fast on its side, but this burns Model A's capacity too. Results ship back over the network.

⚠ Why this feels slow

Multiple queries × a network round trip each = stacked latency. A question that takes 3–5s on a pure import model can take 15–30s+ here, with timeout risk and shallower data sampling (weaker answers). Capacity units are consumed on both models per question. Copilot-first reports belong on a pure import model — or add the tables to Model A directly.
